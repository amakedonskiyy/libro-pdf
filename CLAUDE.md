# CLAUDE.md — libro-pdf

Движок перевода PDF-книг (RU→UK, EN→UK) с сохранением оригинальной вёрстки.
Python 3 + FastAPI + PyMuPDF (fitz) + Tesseract. LLM: Gemini 2.5 Flash (основной), Groq `llama-3.3-70b-versatile` (резерв).

- Прод: https://libro-pdf-production.up.railway.app (Railway, автосборка из main)
- Проверка: `GET /health` → `{"ok": true, "version": ..., "supabase_configured": ...}`
- Фронтенд — отдельный проект в Lovable. В этом репозитории фронтенда НЕТ.

## Файлы

- `main.py` — FastAPI: `/jobs`, `/jobs/{id}`, `/health`, `/preview`, `/cover`; оркестрация задач, `ENGINE_VERSION`
- `pdf_translator.py` — ядро: `extract_blocks`, `build_glossary`, `translate_blocks`, `proofread_blocks`, `build_pdf`, `make_cover`, OCR-конвейер, `_looks_garbled`
- `requirements.txt`, `Dockerfile` — окружение Railway
- `tests/` — смоук-тест (создаётся)

## Железные правила — НЕ НАРУШАТЬ

1. **ENGINE_VERSION бампается при каждом изменении кода** (формат `YYYY-MM-DD-имя-vN` в main.py, отдаётся в `/health`). Изменение без бампа считается незадеплоенным.
2. **Деплой только через git push → Railway.** Фронтенд (Lovable) из этого репо не трогаем. Если меняется API-контракт — сказать владельцу, промпт в Lovable даёт он сам.
3. **`_looks_garbled()` не ослаблять и не «упрощать».** Откалибрована на 0 ложных срабатываний: телефоны, годы, «1990-х», ББК/УДК проходить должны. Любое изменение — только с прогоном проверки на этих случаях.
4. **Двойная защита в `build_pdf` не удаляется:** блок не ставится в PDF, если битым выглядит исходник ИЛИ перевод.
5. **`floor_for` не удаляется:** низ блока не пересекает верх следующего блока; расширение рамки — только для вырожденных (почти нулевых) рамок.
6. **Генерация обложек выключена** (`allow_generate=False`) и не включается. Цепочка обложки: vision-перевод → OCR-на-месте → оставить оригинал. Результат хуже оригинала недопустим.
7. **Ошибки LLM:** 4xx (ключ, модель) — мгновенный fail задачи с понятным сообщением, без ретраев. 5xx/временные — экспоненциальные паузы 3→6→12→24 сек.
8. **Groq-модель только явно:** `llama-3.3-70b-versatile`. Дефолтам клиента не доверять.
9. **Gemini:** `gemini-2.5-flash`, `thinkingBudget: 0` обязателен — без него модель обрезает ответы.
10. **Ключи** не хардкодить, не логировать, не коммитить. Локально — `.env` (в `.gitignore`), на проде — переменные Railway.
11. **Шрифты** — DejaVu Serif/Sans (8 начертаний) через @font-face + `fitz.Archive`. Base-14 не использовать — там нет кириллицы.
12. **Push в origin делает владелец сам** (GitHub Desktop). Коммитить локально можно, push — только по его явной команде.

## Рабочий цикл

1. Перед изменениями: `python tests/smoke.py` — должен быть зелёный.
2. Изменил код → бампнул `ENGINE_VERSION` → смоук снова зелёный → коммит → показать дифф владельцу.
3. После push: Railway собирает 3–5 мин → дождаться ACTIVE → открыть `/health` → убедиться, что `version` новая → тестировать ТОЛЬКО свежим прогоном. Старые результаты из истории сделаны старым кодом.
4. Локально ≠ Railway: баг «34567» (битый cmap шрифта) проявлялся только на сервере. Локальный тест дополняет проверку на проде, а не заменяет её.

## Локальный запуск

```bash
python3 -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
brew install tesseract tesseract-lang   # если не установлен
uvicorn main:app --reload --port 8000
```

## Архитектурные особенности (почему так)

- Украинский длиннее русского на 5–15% → размещение через `page.insert_htmlbox` с автоподбором кегля по ширине И высоте блока.
- Базовый кегль страницы = мода размеров блоков; заголовок = ≥1.4× моды и ≤55 символов (эвристика, калибруется по тестам).
- Маршрутизация «текст vs скан» — по реальному текстовому слою (порог ~60 символов/страницу), НЕ по vision-рецепту: рецепт ошибался.
- Глоссарий (40–60 терминов и имён) строится до перевода и вшивается в системный промпт — обязателен для консистентности по книге.
- Vision-рецепт книги опционален; обычным текстовым книгам не нужен.

## Бэклог (по приоритету)

1. **Обложки (критично):** vision-модуль — Gemini-vision читает обложку → строгий JSON {text, bbox, color, role} → перевод с тем же глоссарием → перерисовка с подбором фона. Честные ограничения: рамки приблизительные, на текстурном фоне возможна видимая заплатка, шрифт не идентичный.
2. Смоук-тест `tests/smoke.py`: 5-страничный PDF, замоканный LLM, < 30 сек, без сети.
3. Авто-фолбэк Gemini→Groq после N подряд 503 — задача доделывается вторым провайдером, а не висит.
4. Фиксы по Волне 1 тестов (отступы/абзацы «полотном», детект заголовков) — после скринов от владельца.
5. Reflow-режим: `POST /jobs/{id}/rebuild` — пересборка чистого PDF с нуля по кнопке (отдельный этап после стабилизации).
6. Серверный запрет параллельных прогонов одной книги + таймаут/потолок стоимости на задачу.
